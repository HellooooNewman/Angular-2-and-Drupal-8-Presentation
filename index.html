What is this?

This is what some call a decoupled or headless drupal. Where the front end(theme) and the backend(content) are completely separate.

Why use headless drupal?

  • Limited by the front-end restrictions of a standard CMS
  • Rich web apps
  • highly custom layouts
  • JavaScript MVC frameworks which don't quite fit into the architecture of a CMS which closely controls how content looks
  • To avoid the dirty hacks of undoing a front-end and amass large technical debt, it's best to have a clean setup from the beginning.
https://www.contentful.com

Use Cases

  • websites and web apps built with MVC-style JavaScript frameworks
  • mobile apps
  • static site generator driven websites(jekyll)
  • cross platform publish sites

How to make your endpoints

Format must always be specified or will default to html

  • REST UI
  • Views Data Export
  • Custom REST Modules

REST UI 


							user/1?_format=json

Views REST Export 


						api/articles?items_per_page=5&offset=1&_format=json
Get Articles

Custom REST Examples 


						entity/node/bundles?_format=json
Get All content Types
